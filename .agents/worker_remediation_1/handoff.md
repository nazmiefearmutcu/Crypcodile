- **UnboundLocalError Regression Bug in `connector.py`**:
  In `src/crypcodile/exchanges/base_onchain/connector.py`, the `_poll_loop` inner try-except block caught exceptions but did not use `continue`. Consequently, when a query failed, execution fell through to step C (`state_payload` construction and queue push), which used variables (like `slot0`, `price`, etc.) that were not initialized or were bound to incorrect values, resulting in `UnboundLocalError`.

- **Flawed Test Assertions in `test_challenger_stress_4.py`**:
  Lines 107-110 and 141-144 in `tests/exchanges/base_onchain/test_challenger_stress_4.py` checked logs for errors via:
  ```python
  assert not any(
      "swaps" in record.message and "local" in record.message
      for record in caplog.records
  )
  ```
  which is flawed because it does not directly query the correct log messages generated by an `UnboundLocalError`.

- **Flawed IPC Reload Test Mocking in `test_challenger_remediation_6.py`**:
  `DummyMockContractFunctions` in `tests/exchanges/base_onchain/test_challenger_remediation_6.py` did not implement `getReserves()`. When testing Aerodrome V2 pools, calls to this function failed with an `AttributeError`.
